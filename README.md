# 날씨 앱

대한민국 행정구역 기반 날씨 정보 조회 앱입니다. 삼성 갤럭시 날씨 앱 스타일의 몰입감 있는 UI를 제공합니다.

## 프로젝트 실행 방법

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npm run dev

# 프로덕션 빌드
npm run build

# 빌드 미리보기
npm run preview
```

### 환경 변수

루트 디렉토리에 `.env` 파일을 생성하고 아래 값을 설정합니다.

```
VITE_OWM_API_KEY=<OpenWeatherMap API Key>
VITE_USE_MOCK=false
```

- `VITE_USE_MOCK=true`로 설정하면 API 호출 없이 목(mock) 데이터로 동작합니다.

## 구현 기능

### 1. 현재 위치 날씨 조회

- 앱 첫 진입 시 브라우저 Geolocation API로 사용자 위치를 감지합니다.
- 위치 권한 거부 시 서울특별시를 기본 위치로 사용합니다.
- 현재 기온, 최저/최고 기온, 체감온도, 습도, 풍속, 일출/일몰, 기압, 가시거리를 표시합니다.
- 현재 날짜, 요일, 시간을 표시하며 1분마다 자동 갱신됩니다.

### 2. 시간대별 예보

- 3시간 간격 최대 8개 시간대의 기온과 날씨 아이콘을 수평 스크롤로 표시합니다.
- 맨 왼쪽에 "지금" 항목으로 현재 날씨를 표시합니다.
- 강수확률(%)이 있는 경우 함께 표시합니다.

### 3. 주간 예보 (5일)

- OWM 5-Day Forecast 데이터를 KST 기준 날짜로 그룹핑하여 일별 예보를 생성합니다.
- 각 날의 최저/최고 기온을 온도 범위 바 차트로 시각화합니다.
- 요일, 날씨 아이콘, 강수확률을 함께 표시합니다.

### 4. 장소 검색

- 대한민국 행정구역(시/도, 구/군, 동) 단위로 검색할 수 있습니다.
- 검색 아이콘 클릭 시 풀스크린 글래스 오버레이 검색창이 열립니다.
- 검색어 입력 시 매칭되는 장소 리스트를 표시하고, 선택 시 해당 위치의 날씨로 전환됩니다.
- 날씨 정보가 없는 경우 "해당 장소의 정보가 제공되지 않습니다." 메시지를 표시합니다.

### 5. 즐겨찾기

- 검색한 장소를 즐겨찾기에 추가/삭제할 수 있습니다 (최대 6개).
- 즐겨찾기 카드에 날씨 상태별 그라데이션 배경과 현재 날씨, 최저/최고 기온을 표시합니다.
- 즐겨찾기 장소의 별칭(이름)을 수정할 수 있습니다.
- 카드 클릭 시 해당 장소의 상세 날씨 페이지로 이동합니다.
- `localStorage`를 활용하여 즐겨찾기 데이터를 영구 저장합니다.

### 6. 대기질 · 미세먼지

- OWM Air Pollution API를 활용하여 PM2.5, PM10 수치를 표시합니다.
- 대기질 등급(좋음/보통/나쁨/매우나쁨/위험)을 색상 코드로 시각화합니다.

### 7. UX 폴리시

- **스켈레톤 로딩**: 날씨 데이터 로딩 시 콘텐츠 영역별 스켈레톤 UI를 표시합니다.
- **에러 카드**: 데이터 로드 실패 시 재시도 버튼이 포함된 에러 카드를 표시합니다.
- **Pull-to-refresh**: 모바일에서 아래로 당겨 날씨 데이터를 새로고침할 수 있습니다.
- **스태거드 애니메이션**: 각 섹션이 순차적으로 슬라이드업 되는 진입 애니메이션을 적용합니다.
- **자동 갱신**: 날씨 데이터 5분, 대기질 데이터 10분 간격으로 자동 갱신됩니다.

### 8. PWA 지원

- 홈 화면에 추가하여 네이티브 앱처럼 사용할 수 있습니다.
- Service Worker를 통한 오프라인 캐싱 (Stale-While-Revalidate)을 지원합니다.
- 오프라인 상태 시 배너를 표시하고 캐시된 데이터를 제공합니다.

### 9. 반응형 디자인

- 375x667 모바일 뷰포트를 기준으로 모바일 퍼스트 설계합니다.
- 데스크탑에서도 자연스러운 레이아웃을 제공합니다 (max-w-lg 컨테이너).
- 즐겨찾기 카드 그리드가 화면 크기에 따라 2열/3열로 전환됩니다.

## 기술적 의사결정 및 이유

### FSD (Feature Sliced Design) 아키텍처

```
src/
├── app/          # 앱 초기화, 프로바이더, 라우팅, 글로벌 스타일
├── pages/        # 라우트 단위 페이지 (Home, Detail)
├── widgets/      # 독립 UI 블록 (WeatherDisplay, SearchBar, FavoriteList, AirQualityCard)
├── features/     # 사용자 인터랙션 (검색, 즐겨찾기 관리, 위치 감지)
├── entities/     # 비즈니스 엔티티 (weather, location)
└── shared/       # 공용 유틸리티, UI 컴포넌트, 설정, 데이터
```

- **레이어 간 단방향 의존성**: 상위 레이어만 하위 레이어를 import하여 의존 관계를 명확히 합니다.
- **슬라이스 단위 모듈화**: 각 기능을 독립적인 슬라이스로 분리하여 유지보수성을 높였습니다.

### TanStack Query

- 날씨 API 응답을 자동 캐싱하여 동일 위치 재조회 시 불필요한 네트워크 요청을 방지합니다.
- `staleTime: 5분`으로 설정하여 합리적인 데이터 신선도를 유지합니다.
- `refetchInterval`로 5분/10분 간격 자동 갱신을 구현하여 항상 최신 데이터를 표시합니다.
- 동일 쿼리 키 기반 캐시 공유로 즐겨찾기 카드에서 중복 API 호출을 방지합니다.

### OpenWeatherMap API 활용 극대화

- Current Weather API의 기존 응답에서 sunrise/sunset, pressure, visibility 등 추가 데이터를 파싱하여 **추가 API 호출 없이** 상세 정보를 확장했습니다.
- 5-Day/3-Hour Forecast의 40개 데이터 포인트를 KST 기준으로 그룹핑하여 주간 예보를 도출합니다.
- Air Pollution API를 별도로 호출하여 미세먼지 데이터를 제공합니다.
- API 호출 예산: 최대 7개 위치 × 3개 엔드포인트 × 5분 간격 ≈ 4.2 calls/min (무료 60 calls/min 내).

### 동적 날씨 배경

- 날씨 상태(맑음/흐림/비/눈/안개)와 시간대(낮/밤)에 따라 풀스크린 그라데이션 배경이 자동으로 전환됩니다.
- CSS transition으로 배경 전환 시 1초 부드러운 애니메이션을 적용합니다.
- 프로스티드 글래스(frosted glass) 스타일의 카드 디자인으로 몰입감 있는 UI를 제공합니다.

### PWA (Progressive Web App)

- `vite-plugin-pwa`를 사용하여 Service Worker와 매니페스트를 자동 생성합니다.
- 날씨 API 응답은 Stale-While-Revalidate, 아이콘은 Cache-First 전략을 적용합니다.
- 오프라인 시에도 캐시된 데이터로 앱을 사용할 수 있습니다.

### 장소 검색

- 제공된 `korea_districts.json` 데이터를 클라이언트 사이드에서 필터링합니다.
- 285개 행정구역(시/도, 구/군, 동)을 포함하며, 이름 부분 일치 검색을 지원합니다.

### Tailwind CSS v4

- CSS-first 설정 방식으로 별도 설정 파일 없이 Vite 플러그인만으로 동작합니다.
- 유틸리티 클래스 기반으로 일관된 디자인 시스템을 빠르게 적용했습니다.
