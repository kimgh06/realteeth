# 날씨 앱

대한민국 행정구역 기반 날씨 정보 조회 앱입니다. 삼성 갤럭시 날씨 앱 스타일의 몰입감 있는 UI를 제공합니다.

**배포 URL**
https://weather.kimgh06.com
---

## 프로젝트 실행 방법

### 환경 변수 설정

루트 디렉토리에 `.env.local` 파일을 생성하고 아래 값을 설정합니다.

```
VITE_OWM_API_KEY=<OpenWeatherMap API Key>
```

- `VITE_USE_MOCK=true`로 설정하면 API 키 없이 목(mock) 데이터로 실행할 수 있습니다.

### 실행

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npm run dev

# 프로덕션 빌드
npm run build

# 빌드 미리보기
npm run preview
```

### 테스트

```bash
npm test          # watch 모드
npm run test:run  # 단회 실행
```

## 구현한 기능

### 1. 현재 위치 날씨 조회

- 앱 첫 진입 시 브라우저 Geolocation API로 사용자의 현재 위치를 자동 감지합니다.
- 위치 권한 거부 시 서울특별시를 기본 위치로 사용합니다.
- **현재 기온**, **당일 최저/최고 기온**, 체감온도, 습도, 풍향·풍속, 기압, 가시거리를 표시합니다.
- 현재 날짜·요일·시각을 표시하며 1분마다 자동 갱신됩니다.

### 2. 날씨 상세 정보

- **시간대별 기온**: 3시간 간격 최대 8개 슬롯(+ 현재 "지금" 포함)의 기온·날씨 아이콘·강수확률 수평 스크롤 표시
- **주간 예보 (5일)**: OWM Forecast 데이터를 KST 기준 날짜로 그룹핑, 일별 최저/최고 기온 범위 바 차트
- **대기질**: PM2.5, PM10 수치 및 AQI 등급(좋음/보통/나쁨/매우나쁨/위험) 색상 시각화
- **일출/일몰 아크**: 현재 시각 기준 태양 위치를 호(arc) 형태로 시각화
- **상세 지표**: 가시거리, 기압

### 3. 장소 검색

- 제공된 `korea_districts.json`을 활용한 **클라이언트 사이드 검색**입니다.
- 시·도, 구·군, 동 등 **모든 행정 단위**로 검색 가능합니다. (예: "서울특별시", "종로구", "청운동")
- 검색어 입력 시 150ms 디바운스 후 매칭되는 **장소 목록을 표시**하고, 목록에서 선택할 수 있습니다.
- 날씨 정보가 제공되지 않는 경우 **"해당 장소의 정보가 제공되지 않습니다."** 메시지를 표시합니다.
- 최근 검색어를 저장하여 빠른 재접근이 가능합니다.
- `/` 키보드 단축키로 검색창을 즉시 열 수 있습니다.

### 4. 즐겨찾기

- 현재 조회 중인 장소를 별 아이콘으로 즐겨찾기에 **추가/삭제**할 수 있습니다.
- **최대 6개**까지 추가 가능하며, 초과 시 토스트 알림으로 안내합니다.
- 즐겨찾기 카드에는 날씨 상태별 그라데이션 배경과 함께 **현재 날씨, 당일 최저/최고 기온**을 표시합니다.
- 카드 클릭 시 해당 장소의 **상세 날씨 페이지**로 이동합니다.
- 즐겨찾기 장소의 **별칭(이름)을 수정**할 수 있습니다.
- 삭제 후 5초 내 **실행 취소** 토스트 버튼으로 복원할 수 있습니다.
- `localStorage`를 활용하여 즐겨찾기 데이터를 영구 저장합니다.

### 5. 사이드바 네비게이션

- 좌측 슬라이드 사이드바에서 즐겨찾기 목록을 확인하고 빠르게 이동할 수 있습니다.
- 현재 위치로 돌아가거나 검색창을 바로 열 수 있습니다.

### 6. UX / 반응형

- **반응형 디자인**: 모바일(375px 기준)·데스크탑 모두 지원, 즐겨찾기 그리드 2열/3열 전환
- **동적 배경**: 날씨 상태(맑음/흐림/비/눈/안개)와 낮·밤에 따라 배경 그라디언트가 자동 전환됩니다.
- **스켈레톤 로딩**: 데이터 로딩 중 레이아웃 시프트 없는 스켈레톤 UI
- **에러 카드**: 데이터 로드 실패 시 재시도 버튼 제공
- **Pull-to-refresh**: 모바일에서 아래로 당겨 날씨 데이터를 새로고침
- **자동 갱신**: 날씨 5분 · 대기질 10분 간격
- **PWA**: 홈 화면 추가 및 Service Worker 오프라인 캐싱 지원
- **오프라인 배너**: 네트워크 연결 끊김 시 안내 표시

## 기술적 의사결정 및 이유

### FSD (Feature Sliced Design) 아키텍처

```
src/
├── app/          # 앱 초기화, Provider, 라우팅, 글로벌 스타일
├── pages/        # 라우트 단위 페이지 (Home, Detail, Favorites)
├── widgets/      # 독립 UI 블록 (WeatherDisplay, SearchBar, Sidebar, AirQualityCard, SunArcCard)
├── features/     # 사용자 인터랙션 (위치 감지, 장소 검색, 즐겨찾기 관리)
├── entities/     # 비즈니스 엔티티 (weather, location)
└── shared/       # 공용 유틸리티, UI 컴포넌트, 설정, 검색 인덱스
```

레이어 간 단방향 의존성(`app → pages → widgets → features → entities → shared`)을 강제해 변경의 영향 범위를 명확히 격리합니다. 기능 추가·수정 시 관련 슬라이스만 건드리면 됩니다.

### 클라이언트 사이드 장소 검색

`korea_districts.json`을 앱 번들에 포함해 별도 검색 서버 없이 즉각적인 응답을 제공합니다. 세그먼트 기반 점수 알고리즘(완전 일치 100점 → 부분 포함 50점)으로 관련도 순 정렬을 구현하고, TTL 30초 인메모리 캐시로 반복 검색 성능을 최적화했습니다.

### TanStack Query 데이터 관리

- `queryKey: ["weather", lat, lon]`으로 위치별 날씨를 독립적으로 캐싱합니다.
- `refetchInterval` 5분/10분으로 자동 갱신을 구현해 항상 최신 데이터를 유지합니다.
- 동일 쿼리 키를 공유하므로 즐겨찾기 카드에서 중복 API 호출이 발생하지 않습니다.
- `select` 옵션으로 즐겨찾기 카드에서 요약 데이터만 구독합니다(`useWeatherSummary`).
- 날씨와 대기질 쿼리를 분리해 일부 실패 시 나머지 정보는 정상 표시됩니다.

### API 키 보안 (프록시 구조)

`/api/owm/*` 경로를 Vite 개발 서버 프록시로 OpenWeatherMap에 포워딩합니다. API 키가 클라이언트 번들에 노출되지 않으며, 프로덕션 배포 시 동일한 구조로 서버사이드 프록시를 구성합니다.

### 즐겨찾기 Soft Delete (실행 취소)

삭제 요청 시 상태에서 즉시 제거하되 5초간 `pendingRef`에 보관합니다. 타임아웃 만료 전 "실행 취소" 클릭 시 원래 인덱스에 복원합니다. 확인 다이얼로그 없는 빠른 조작과 안전망을 동시에 제공합니다.

### OpenWeatherMap API 활용

- Current Weather API 단일 응답에서 sunrise/sunset, pressure, visibility 등 추가 데이터를 파싱해 **추가 호출 없이** 상세 정보를 확장합니다.
- 5-Day/3-Hour Forecast의 40개 포인트를 KST 기준 날짜로 그룹핑해 주간 예보를 도출합니다.
